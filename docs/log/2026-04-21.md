## `_LOADED_AT` fix (closes ADR-0010 known limitation)

- [x] Decided to keep the `_LOADED_AT` column rather than drop it. 18 dbt
  staging models already select it ([stg_main_book__valuations.sql:31](../../dbt/models/staging/main_book/stg_main_book__valuations.sql) and siblings); dropping would force 18 model edits for no upside.
- [x] Switched the pipe COPY in [`snowflake_snowpipe/main.tf`](../../terraform/modules/snowflake_snowpipe/main.tf) from the implicit positional form to a transformed form: `COPY INTO tbl(col1, ..., colN, _LOADED_AT) FROM (SELECT $1, ..., $N, CURRENT_TIMESTAMP() FROM @stage)`. Templated in HCL via `for i in range(length(each.value.columns)) : format("$%d", i + 1)` — Terraform `format()` produces the literal `$1, $2, ...` string without HCL interpolation collisions.
- [x] Fixed the lying README on the `snowflake_snowpipe` module — it claimed `_SOURCE_FILE` and `_SOURCE_ROW` columns existed; only `_LOADED_AT` does.
- [x] Apply replaced all 18 pipes (ForceNew on `copy_statement`, same precedent as ADR-0010 §"AUTO_INGEST behaviour"). Landing tables and the notification integration are unchanged.
- [x] Backfilled 1.8M existing rows to non-null `_LOADED_AT = current_timestamp()` via a one-shot Python script (deleted after run). Spot-check on `main_book_valuations` confirms 100K/100K rows non-null.

## Snowpipe quarantine (ADR-0012)

- [x] Wrote ADR-0012 picking option 2 (one shared table, one task using `VALIDATE_PIPE_LOAD()`) over per-pipe tables, stream-based capture, dynamic stored procs, and Event Grid DLQ. Primary reason: matches ADR-0009's "shared at this scale" precedent; cross-source triage is a single `WHERE pipe_name = …`.
- [x] New module [`snowflake_quarantine`](../../terraform/modules/snowflake_quarantine/) — `PIPE_ERRORS` table (mirrors `VALIDATE_PIPE_LOAD()` columns + `pipe_name` + `captured_at`; renames the reserved `CHARACTER` to `CHARACTER_POS`) plus `TSK_CAPTURE_PIPE_ERRORS` Snowflake Task. Task is owner-mode (runs as ACCOUNTADMIN), runs on `LOAD_WH`, every 5 min, MERGE-keyed on `(pipe_name, file_name, row_number, line)` with `COALESCE` on each key.
- [x] Body of the task is one big UNION ALL across all 18 pipe FQNs — generated by a Terraform `for` loop. Snowflake renders the full ~190-line MERGE on `show tasks`; verified.
- [x] Added `RAW_QUARANTINE` schema (created inline in `dev/main.tf`, not in `database_layers` — it spans tenants). Wired into `module.rbac.schemas` so the existing access-role machinery generates `AR_…_RAW_QUARANTINE_RW/RO`.
- [x] `FR_ENGINEER` gains RW; `FR_ANALYST` gains RO (data-quality visibility — analysts should see what failed).
- [x] Cost: ~0.05–0.1 credit/month on `LOAD_WH`. Inside the existing `RM_LOAD_WH` 5-credit cap.
- [x] Verified post-apply: task `state = started`, next run scheduled in ~5 min, definition references all 18 pipe FQNs.
