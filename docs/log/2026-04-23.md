## ADR-0016 — stacked-PR merge convention

- [x] Wrote [ADR-0016](../adr/0016-stacked-pr-merge-convention.md). Decision:
  **retarget downstream PR base to `master` BEFORE deleting the upstream
  branch.** Four-line `gh` recipe documented as canonical. Stacked-PR
  tooling (`spr`, Graphite) explicitly deferred until stack depth or
  frequency justifies it.
- [x] Cross-referenced under a new "GitHub workflow" section in
  [issues-and-fixes.md](issues-and-fixes.md) so anyone replicating the
  project hits the recipe without reading the ADR.
